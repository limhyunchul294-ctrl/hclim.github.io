# 커밋 메시지 템플릿
# 변경 내용에 맞게 적절한 한글 메시지를 작성하세요

## 커밋 메시지 작성 가이드

변경 내용에 따라 적절한 접두사를 사용하세요:

- **Feat**: 새로운 기능 추가
- **Fix**: 버그 수정 및 오류 해결
- **Docs**: 문서 수정 또는 추가
- **Style**: 코드 포맷팅, 스타일 변경
- **Refactor**: 코드 리팩토링
- **Config**: 설정 파일 변경
- **Test**: 테스트 코드 추가/수정
- **Chore**: 빌드 업무 수정, 패키지 매니저 설정 등

## 예시

- Feat: 사용자 인증 기능 추가
- Fix: 로그인 오류 수정
- Docs: API 사용법 문서 추가
- Refactor: 코드 구조 개선 및 최적화
